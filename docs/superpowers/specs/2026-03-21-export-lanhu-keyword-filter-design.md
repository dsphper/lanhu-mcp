# Export Lanhu Skill - 关键词过滤功能设计规格

## 概述

为 export-lanhu skill 添加智能关键词过滤功能，支持两阶段交互式导出：先预览匹配结果，用户确认后再执行导出。

## 需求背景

- **目标**：用户可以通过自然语言描述导出特定类型的设计数据
- **场景**：只导出"登录"相关页面、只导出"支付"相关设计等
- **约束**：保持向后兼容，不指定关键词时行为不变

## Python 脚本参数设计

### 现有参数

| 参数 | 说明 |
|------|------|
| `URL` | 蓝湖 URL（必填） |
| `--output, -o` | 自定义输出目录 |
| `--no-slices` | 跳过切图下载 |
| `--no-preview` | 跳过预览图下载 |

### 新增参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `--list` | flag | 只列出设计，不下载，输出 JSON 到 stdout |
| `--keyword KEYWORD` | string | 过滤关键词（逗号分隔多个，OR 逻辑） |
| `--keywords K1 K2 ...` | list | 过滤关键词（空格分隔，支持多个参数） |
| `--ids ID1,ID2,ID3` | string | 指定设计 ID 导出（精确控制，优先级最高） |

### 参数组合规则

| 组合 | 行为 |
|------|------|
| `--list` | 输出全部设计列表 JSON |
| `--list --keyword 登录` | 输出过滤后的设计列表 JSON |
| `--keyword 登录` | 直接导出匹配的设计（无预览） |
| `--ids id1,id2` | 精确导出指定 ID 的设计 |

## --list 输出格式

```json
{
  "source_url": "https://lanhuapp.com/...",
  "project_name": "电商APP设计",
  "project_id": "xxx-xxx-xxx",
  "total": 28,
  "matched_count": 5,
  "matched": [
    {"id": "abc123", "name": "登录页"},
    {"id": "def456", "name": "登录页-手机号"},
    {"id": "ghi789", "name": "登录页-验证码"},
    {"id": "jkl012", "name": "登录成功"},
    {"id": "mno345", "name": "忘记登录密码"}
  ],
  "unmatched_count": 23,
  "unmatched": [
    {"id": "xxx", "name": "商品详情"},
    {"id": "yyy", "name": "支付成功"}
  ]
}
```

**输出说明：**
- `matched`：匹配关键词的设计列表
- `unmatched`：未匹配的设计列表（当数量超过 10 时，只返回前 10 个作为示例）
- 所有输出到 stdout，日志信息输出到 stderr

## 关键词匹配规则

| 规则 | 说明 |
|------|------|
| 匹配方式 | 包含匹配（设计名包含关键词即匹配） |
| 多关键词逻辑 | OR（匹配任意一个即可） |
| 大小写 | 不区分 |
| 分隔符 | 支持逗号分隔或空格分隔 |

**示例：**
- `--keyword 登录,注册` → 匹配包含"登录"或"注册"的设计
- `--keywords 登录 注册 忘记密码` → 匹配包含"登录"或"注册"或"忘记密码"的设计

## 使用方式

用户可以通过自然语言描述导出需求：

- "导出蓝湖 <URL> 和登录相关的页面数据"
- "导出蓝湖 <URL> 的全部设计"
- "导出蓝湖 <URL> 中登录、注册相关的设计"

## 前置条件

1. 在项目根目录创建 `.claude/lanhu.config.json` 配置文件
2. 配置文件中必须包含 `cookie` 字段

## 执行流程

### 步骤 1：解析用户输入

1. **提取蓝湖 URL**
   - 匹配 `lanhuapp.com` 开头的链接
   - 支持 `http://` 和 `https://`

2. **提取关键词**
   - 从用户描述中识别业务词（如"登录"、"注册"、"支付"）
   - 常见关键词模式：
     - "和 X 相关的"
     - "X 相关的页面"
     - "包含 X 的设计"
   - 如果用户说"全部"、"所有"或未指定关键词，则无关键词

3. **多关键词处理**
   - 多个关键词用逗号连接传递给脚本

### 步骤 2：获取设计列表（预览）

执行命令获取设计列表：

```bash
cd skills/export-lanhu/scripts
python export_lanhu.py "<URL>" --keyword <关键词> --list
```

无关键词时：

```bash
python export_lanhu.py "<URL>" --list
```

### 步骤 3：展示结果并确认

解析 JSON 输出，展示给用户：

```
🎯 匹配"登录"的设计图 (5/28)：

  ✅ 登录页
  ✅ 登录页-手机号
  ✅ 登录页-验证码
  ✅ 登录成功
  ✅ 忘记登录密码

📋 其他设计（23 个）：商品详情、支付成功、我的...

是否导出这 5 个设计？
```

**交互选项：**
- 用户确认 → 执行导出
- 用户要求调整 → 重新过滤或使用 `--ids` 精确指定

### 步骤 4：执行导出

用户确认后执行：

```bash
python export_lanhu.py "<URL>" --keyword <关键词>
```

精确指定 ID：

```bash
python export_lanhu.py "<URL>" --ids <id1>,<id2>,<id3>
```
